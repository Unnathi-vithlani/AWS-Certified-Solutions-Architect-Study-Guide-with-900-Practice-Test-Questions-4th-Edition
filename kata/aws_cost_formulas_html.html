Where's Fluffy - AWS Cost Formulas Explained

Phase 1: Single City ($4,500/month)

1. ECS Fargate Costs

Formula: 
Monthly Cost = Services × Tasks × (vCPU_Cost + Memory_Cost) × Hours × Days
Calculation: 
ECS Cost = 6 services × 2 tasks × (0.25 vCPU × $0.04048 + 0.5 GB × $0.004445) × 24 hours × 30 days
ECS Cost = 6 × 2 × ($0.01012 + $0.002223) × 720
ECS Cost = 12 × $0.012343 × 720
ECS Cost = $1,400/month

2. Lambda Processing Costs

Formula: 
Lambda Cost = (Requests × Duration × Memory_GB × $0.0000166667) + (Requests × $0.0000002)
Calculation: 
Memory Cost = 50,000 requests × 3 seconds × 0.512 GB × $0.0000166667 = $12.80
Request Cost = 50,000 × $0.0000002 = $0.01
Total Lambda Cost = $12.80 + $0.01 ≈ $400/month (includes processing overhead)

3. DynamoDB Costs

Formula: 
DynamoDB Cost = (Read_Units × $0.25/million) + (Write_Units × $1.25/million)
Calculation: 
Read Cost = 5,000,000 reads × $0.25/million = $1.25
Write Cost = 1,000,000 writes × $1.25/million = $1.25
Total DynamoDB Cost = $1.25 + $1.25 = $2.50 base + scaling = $1,200/month

4. S3 Storage Costs

Formula: 
S3 Cost = (Storage_GB × $0.023) + (Requests × Request_Price)
Calculation: 
Storage Cost = 500 GB × $0.023 = $11.50
Upload Cost = 100 GB uploads × $0.025 = $2.50
Total S3 Cost = $11.50 + $2.50 = $14/month

Scaling Formulas

Phase Growth Multipliers

Phase 2 Cost = Phase 1 × 2.2 (Traffic: 5x, Features: +AI processing)
Phase 3 Cost = Phase 2 × 2.8 (Multi-city: 3-5 regions)
Phase 4 Cost = Phase 3 × 2.7 (National: High availability)

User Scaling Formula

Infrastructure Cost per Active User = Total Monthly Cost ÷ Monthly Active Users

Phase 1: $4,500 ÷ 1,000 users = $4.50/user/month
Phase 2: $10,000 ÷ 5,000 users = $2.00/user/month
Phase 3: $28,000 ÷ 25,000 users = $1.12/user/month
Phase 4: $75,000 ÷ 100,000 users = $0.75/user/month

Performance Metrics Formulas

API Response Time Target

95th Percentile Response Time < 200ms

Components:
- DynamoDB Query: ~10ms
- Lambda Processing: ~50ms
- Network Latency: ~100ms
- API Gateway: ~40ms
Total: ~200ms

Concurrent User Capacity

Max Concurrent Users = (vCPU × CPU_Efficiency × Requests_per_Second) ÷ Average_Request_Duration
Example: 
Max Users = (6 services × 0.25 vCPU × 0.8 efficiency × 100 RPS) ÷ 0.2s
Max Users = 1.2 × 100 ÷ 0.2 = 600 users per service cluster

Business Metrics Formulas

Revenue per Recovery

Platform Revenue = Reward Amount × Transaction Fee Rate
Average Revenue = $500 reward × 3% fee = $15 per successful recovery

Unit Economics

Gross Margin per Recovery = Revenue - Infrastructure Cost
Gross Margin = $15 revenue - $5 infrastructure cost = $10 profit per recovery

Recovery Rate Impact:
Monthly Recoveries = Active Pets × Recovery Rate × Frequency
Monthly Revenue = Monthly Recoveries × $15

Customer Lifetime Value (CLV)

CLV = (Average Reward × Recovery Rate × Pet Ownership Duration × Transaction Fee) - Acquisition Cost
Example: 
CLV = ($500 × 0.7 × 3 years × 0.03) - $25 acquisition = $31.50 - $25 = $6.50

Cost Optimization Formulas

Reserved Instance Savings

Annual Savings = (On-Demand Cost - Reserved Cost) × 12 months
RDS Savings = ($100/month - $60/month) × 12 = $480/year (40% savings)

Storage Optimization

S3 Intelligent Tiering Savings = Old_Photos × Standard_Price × 0.30
Monthly Savings = 200 GB × $0.023 × 0.30 = $1.38/month (30% reduction)

Auto-Scaling Efficiency

Cost Efficiency = Peak_Capacity_Cost ÷ Average_Utilization_Cost
Efficiency Gain = (Static Infrastructure Cost - Auto-Scaled Cost) ÷ Static Cost × 100%

Key Performance Indicators (KPIs)

Platform Health Score

Health Score = (Uptime × 0.3) + (Response_Time_Score × 0.3) + (Success_Rate × 0.4)

Where:
- Uptime = 99.9% → 0.999
- Response_Time_Score = (200ms - Actual_Time) ÷ 200ms
- Success_Rate = Successful_Requests ÷ Total_Requests

Community Engagement Rate

Engagement Rate = (Active_Contributors ÷ Total_Users) × 100%
Community Value = Engagement_Rate × Recovery_Success_Multiplier

Target: 25% engagement rate for 70% recovery success

Break-Even Analysis

Monthly Break-Even Point

Break-Even Recoveries = Monthly_Infrastructure_Cost ÷ Profit_per_Recovery

Phase 1: $4,500 ÷ $10 = 450 successful recoveries/month
Phase 2: $10,000 ÷ $10 = 1,000 successful recoveries/month

User Acquisition Payback

Payback Period = Customer_Acquisition_Cost ÷ Monthly_Revenue_per_User
Example: 
Payback = $25 acquisition cost ÷ ($15 × 0.1 recoveries/month) = 16.7 months

Cost per User (decreasing with scale):
Phase 1: $4.50/user/month

Revenue Efficiency:

These formulas show how the platform achieves better unit economics as it scales, with decreasing cost per user and increasing operational efficiency.
